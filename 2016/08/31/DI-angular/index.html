Angular 三种依赖注入的方式

2016年8月31日

Dependency Injectio

所谓依赖注入，在我看来，即需要什么就注入到相应的Function中，“饭来张口”，就是典型的依赖注入模式。

Angular中的依赖注入有三种方式： 推断式注入，内联式注入，标记式注入

推断式注入：

angular.module('app',[])
    .controller('ctr',function ($scope){

        $scope.some = xx;    //业务逻辑
    });

推断式注入比较简洁 ，要求参数名称($scope)必须与注入的服务名称相同。但是存在弊端，经过代码压缩后就会报错，代码压缩工具会替换变量名称，会把注入的$scope替换掉。

内联式注入：

angular.module('app',[])
    .controller('ctr',['$scope',function ($scope){

    }])

内联式注入需要传入一个数组，数组的最后一个参数是真正的方法体，需要注意的是，方法体函数参数与数组内的参数顺序一致。使用内联式注入就可以解决代码压缩导致无法运行的问题，参数只要保证顺序一致，经过压缩后也不会出现问题。

标记式注入：

var app = angular.module('app',[]);

var ctrFn = function ($scope){
};

ctrFn.$injector = ['$scope'];
app.controller('ctr',ctrFn);

标记式注入与内联式注入一样，要求参数顺序一致，并且也可以解决代码压缩问题。
